# 🎯 FactCCX 幻觉检测框架（完整流程文档）

---

# Step 0：输入定义

### 输入

```text
D = source document（原文）
S = summary（生成摘要）
```

---

### 预处理

```text
S → 句子切分 → {s₁, s₂, ..., sₙ}
```

👉 FactCCX 的检测单位是：

> **单个 summary sentence（claim）**

---

### 输出（逐句）

```json
{
  "sentence": s_i,
  "label": "CONSISTENT / INCONSISTENT",
  "source_span": "...",
  "claim_span": "..."
}
```

---

# Step 1：构造检测单元

### 输入

```text
D
s_i
```

---

### 构造

```text
x_i = (D, s_i)
```

👉 每一句单独与整篇 document 做匹配

---

# Step 2：BERT 输入构造

### 输入

```text
x_i = (D, s_i)
```

---

### 构造序列

```text
[CLS] D_tokens [SEP] s_i_tokens [SEP]
```

---

### 关键实现细节

#### 1️⃣ 长度限制

```text
max_seq_length = 512
```

处理策略：

```text
优先保留 s_i（claim）
截断 D（document）
```

---

#### 2️⃣ segment 标记

```text
token_type_ids:
    0 → document
    1 → claim
```

---

### 输出

```text
input_ids
attention_mask
token_type_ids
```

---

# Step 3：语义编码（BERT）

### 输入

```text
input_ids
```

---

### 处理

```text
H = BERT(D, s_i)
```

---

### 输出

```text
H = {h_cls, h₁, h₂, ..., h_T}
```

---

# Step 4：一致性判别（分类头）

### 输入

```text
h_cls
```

---

### 计算

```text
logits = W * h_cls + b
prob = softmax(logits)
```

---

### 输出

```text
label ∈ {CONSISTENT, INCONSISTENT}
```

---

# Step 5：原文证据定位（source span）

### 输入

```text
H（所有 token 表示）
```

---

### 计算

```text
start_logits = W_s * H
end_logits   = W_e * H
```

---

### 解码

```text
start = argmax(start_logits)
end   = argmax(end_logits)
```

---

### 输出

```text
source_span = D[start:end]
```

---

### 含义

👉 模型认为：

> “这段原文最能支持当前判断”

---

# Step 6：错误位置定位（claim span）

### 输入

```text
H（token 表示）
```

---

### 计算（只针对 claim 部分）

```text
start_logits_claim
end_logits_claim
```

---

### 解码

```text
start = argmax
end   = argmax
```

---

### 输出

```text
claim_span = s_i[start:end]
```

---

### 含义

👉 当 label = INCONSISTENT 时：

> “这句话错在这里”

---

# Step 7：结果输出

### 整合

```json
{
  "sentence": s_i,
  "label": "INCONSISTENT",
  "source_span": "...原文证据...",
  "claim_span": "...错误位置..."
}
```

# 🧠 完整运行示例

---

## 输入

```text
D:
Dr. Alan Reed returned to work on Tuesday.

s₁:
Mary Johnson returned to work on Tuesday.
```

---

## Step 1

```text
(D, s₁)
```

---

## Step 2

```text
[CLS] Dr Alan Reed returned ... [SEP] Mary Johnson returned ... [SEP]
```

---

## Step 3

BERT 编码 → 得到 hidden states

---

## Step 4

```text
label = INCONSISTENT
```

---

## Step 5

```text
source_span = "Dr. Alan Reed returned to work"
```

---

## Step 6

```text
claim_span = "Mary Johnson"
```

---

## Step 7（最终输出）

```json
{
  "sentence": "Mary Johnson returned to work on Tuesday.",
  "label": "INCONSISTENT",
  "source_span": "Dr. Alan Reed returned to work",
  "claim_span": "Mary Johnson"
}
```

# ⚠️ 关键注意事项（与你实验强相关）

---

## ❶ 检测粒度

```text
单位 = 单句（sentence-level）
```

不是 summary-level

---

## ❷ 输入结构

```text
(document, sentence)
```

不是：

```text
(sentence, sentence)
```

---

## ❸ 输出结构

FactCCX =

```text
classification + explanation
```

---

## ❹ 不包含 error 类型分类

不会输出：

```text
entity error / number error
```

---

## ❺ 长文截断是核心实现点

```text
512 token 限制 → 必须截断 document
```

---

# 🔥 最终一句总结

> **FactCCX 是一个 document-level 的句子级事实验证模型：对每个摘要句子进行一致性判别，并同时定位原文中的证据片段与句子中的错误片段，从而实现可解释的幻觉检测。**
