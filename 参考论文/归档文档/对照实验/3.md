
Title

RAGTruth: A Hallucination Detection Benchmark for Retrieval-Augmented Generation

Year

2024

Venue

ACL Findings 2024

Task

RAG-based QA / 长文本回答的幻觉检测

Core Idea

利用 retrieval证据 + claim级别对齐，逐条判断回答中的内容是否被检索文档支持。

👉 本质：evidence-grounded verification pipeline

Framework Type

✅ RAG-based

Pipeline（核心步骤）
Step 1: Answer → Claim Decomposition
输入：LLM生成的answer
操作：拆分为 atomic claims

例子：

Answer:
The patient has diabetes and takes insulin.

→ Claims:
C1: patient has diabetes
C2: patient takes insulin
Step 2: Evidence Retrieval
使用 query / answer → retrieval
得到 supporting documents

👉（关键点：不是直接验证，而是先找证据）

Step 3: Claim–Evidence Matching

对每个 claim：

是否有 evidence 支持？
支持强度？

输出：

supported
unsupported
contradictory
Step 4: Aggregation

将 claim-level 结果汇总为：

hallucination score
factual consistency score
Input / Output
Input
question
answer
retrieved documents
Output
claim-level labels
overall hallucination score
