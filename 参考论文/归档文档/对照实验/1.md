Asking and Answering Questions to Evaluate the Factual Consistency of Summaries
https://aclanthology.org/anthology-files/anthology-files/pdf/acl/2020.acl-main.450.pdf

# 一、整体流程（你先建立全局结构）

你实现的就是这个 pipeline：

```
(summary, source)
      ↓
[1] 抽取答案候选（从 summary）
      ↓
[2] 用 QG 生成问题（基于 answer + source）
      ↓
[3] 过滤问题
      ↓
[4] 对每个问题：
        用 QA 在 summary 上答
        用 QA 在 source 上答
      ↓
[5] 比较两个答案（F1）
      ↓
[6] 平均 → QAGS 分数
```

---

# 二、从 0 开始：输入是什么

你需要的数据结构是：

```
source_text  = 原文（长文本）
summary_text = 模型生成摘要（短文本）
```

👉 每一对 (source, summary) 单独跑一次 QAGS

---

# 三、Step 1：从 summary 抽取“答案候选”

这是第一步，也是最关键的一步。

## 1.1 用什么工具

论文用：

* spaCy
* 模型：`en_core_web_sm`

## 1.2 抽什么

从 **summary_text** 里抽：

* Named Entities（人名、地点、组织等）
* Noun Phrases（名词短语）

👉 合在一起作为候选答案

## 1.3 数量控制

最终只保留：

```
最多 10 个 answer candidates
```

规则：

* 多于 10 → 随机采样到 10
* 少于 10 → 随机重复补到 10

---

## 1.4 示例

### summary：

```
"Faisal Khan attacked people at Cambridge University on Friday afternoon."
```

抽出来可能是：

```
Faisal Khan
Cambridge University
Friday afternoon
people
```

假设只有 4 个 → 复制到 10 个

---

# 四、Step 2：生成问题（Question Generation）

## 2.1 用什么模型

论文用：

* BART（seq2seq）
* 用 fairseq 训练
* 数据：NewsQA

👉 你可以直接用 HuggingFace 的 QG 模型代替（复现足够）

---

## 2.2 输入格式（非常关键）

模型输入不是单纯 summary

而是：

```
[answer] + [SEP] + [source_text]
```

👉 模型任务：

> “围绕这个 answer，在这个文章上下文里，生成一个问题”

---

## 2.3 每个 answer 生成多少问题

设置：

```
beam_size = 10
```

所以：

```
10 answers × 10 = 100 个问题
```

---

## 2.4 解码参数

必须对齐论文：

```
beam search
beam size = 10
length_penalty = 1.0
min_length = 8
max_length = 60
trigram_blocking = True
```

---

## 2.5 示例

对于 answer：

```
answer = "Faisal Khan"
```

生成问题：

```
"What is the attacker's name?"
"Who carried out the attack?"
"Who was responsible for the incident?"
...
```

---

# 五、Step 3：过滤问题（非常重要）

你不能直接用 100 个问题。

---

## 3.1 规则过滤

删除：

* 重复问题
* 长度 < 3 token
* “?” 后面的垃圾文本

---

## 3.2 QA 过滤（关键）

对每个问题：

👉 用 QA 模型在 **summary 上回答**

如果：

```
模型输出 = no answer
```

👉 删除这个问题

---

## 3.3 保留数量

最终：

```
K = 20 个问题
```

如果不够：

👉 从剩下的里随机补到 20

---

# 六、Step 4：QA（核心检测步骤）

## 4.1 用什么模型

论文用：

* BERT-large (uncased)
* fine-tuned on SQuAD 2.0

👉 你可以用：

```
deepset/bert-large-uncased-whole-word-masking-squad2
```

---

## 4.2 对每个问题做两次 QA

对于每个 question q：

### ① 在 summary 上回答

```
a_summary = QA(summary_text, q)
```

### ② 在 source 上回答

```
a_source = QA(source_text, q)
```

---

## 4.3 示例

问题：

```
"What is the attacker's name?"
```

结果：

```
a_summary = "Faisal Khan"
a_source  = "Usman Khan"
```

---

# 七、Step 5：计算答案相似度（核心 scoring）

论文用：

```
token-level F1
```

---

## 5.1 F1 定义

```
precision = overlap_tokens / tokens_in_summary_answer
recall    = overlap_tokens / tokens_in_source_answer

F1 = 2 * precision * recall / (precision + recall)
```

---

## 5.2 示例

### case 1（完全错）

```
Faisal Khan
Usman Khan
```

overlap = ["Khan"]

```
precision = 1/2
recall = 1/2
F1 = 0.5
```

👉 注意：不是 0！

---

### case 2（完全一致）

```
Six matches
Six matches
```

F1 = 1.0

---

### case 3（完全不重合）

```
Cambridge University
Fishmongers Hall
```

F1 = 0

---

# 八、Step 6：计算最终 QAGS

对于 20 个问题：

```
QAGS = 平均(F1_1, F1_2, ..., F1_20)
```

---

# 九、完整例子（从头跑一遍）

我给你一个最清晰的 end-to-end：

---

## 输入

### source：

```
Usman Khan attacked people at Fishmongers Hall in London on Friday.
```

### summary：

```
Faisal Khan attacked people at Cambridge University on Friday afternoon.
```

---

## Step 1：抽 answer

```
Faisal Khan
Cambridge University
Friday afternoon
```

---

## Step 2：生成问题（示例）

```
Q1: What is the attacker's name?
Q2: Where did the attack happen?
Q3: When did the attack happen?
```

---

## Step 3：QA

| 问题 | summary 答案           | source 答案        |
| -- | -------------------- | ---------------- |
| Q1 | Faisal Khan          | Usman Khan       |
| Q2 | Cambridge University | Fishmongers Hall |
| Q3 | Friday afternoon     | Friday           |

---

## Step 4：算 F1

| 问题 | F1  |
| -- | --- |
| Q1 | 0.5 |
| Q2 | 0   |
| Q3 | 0.5 |

---

## Step 5：平均

```
QAGS = (0.5 + 0 + 0.5) / 3 = 0.33
```

👉 分数低 → 幻觉多

# 十、你复现时最容易踩的坑（非常重要）

我帮你提前踩好：

---

## ⚠️ 1. 一定要从 summary 抽 answer

不是 source！

否则你检测的就变成：

👉 “summary 有没有覆盖 source”

而不是：

👉 “summary 有没有编造”

---

## ⚠️ 2. QG 输入必须带 answer

不是：

```
input = summary
```

而是：

```
input = answer + source
```

---

## ⚠️ 3. 一定要做 QA-based filtering

否则：

* 很多垃圾问题
* 噪声极大

---

## ⚠️ 4. QA 必须支持 no-answer

必须用 SQuAD2.0 类型模型

否则你无法过滤幻觉问题

---

## ⚠️ 5. 不要用 semantic similarity

论文用的是：

```
token F1
```

不是 embedding similarity

---

## ⚠️ 6. 问题数量要 ≥20

否则不稳定

---

# 十一、最小可复现版本（你可以直接写代码）

如果你想快速复现一个简化版：

### 用：

* spaCy → 抽实体
* T5/BART → QG
* HuggingFace QA → SQuAD2 model

### 流程：

```
for each (source, summary):

    answers = extract_entities(summary)[:10]

    questions = []
    for ans in answers:
        questions += generate_questions(ans, source)

    questions = filter(questions)[:20]

    scores = []
    for q in questions:
        a_s = QA(summary, q)
        a_x = QA(source, q)

        f1 = compute_f1(a_s, a_x)
        scores.append(f1)

    QAGS = mean(scores)
```
