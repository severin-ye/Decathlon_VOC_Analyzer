# MVP与迭代计划

## MVP 阶段

目标是先把系统闭环跑通，而不是一次做完最复杂版本。

### MVP 范围

- 商品图 + 商品文案统一索引
- 评论做 aspect 级结构化抽取
- 评论驱动商品证据召回
- 按 product_id 聚合
- 输出综合意见与改进建议
- 全流程保留 evidence id

### MVP 不做的事

- 图像区域级裁剪
- 复杂客户画像聚类
- 时间趋势分析
- 多轮 agent 反思

## Phase 2

- 加入商品印象模型与客户印象模型
- 强化聚合统计与人群差异分析
- 引入 reranker 精排

## Phase 3

- 加入图像区域级证据
- 引入评论与图片局部区域对齐
- 生成更细粒度的改进建议

## 实施顺序的核心理由

对话中明确建议不要一上来就做最复杂的 region-level 原始模态检索，而应先做稳定的商品图 + 商品文案统一召回闭环。
