# 改进建议生成

## 本层目标

在综合意见的基础上，输出可解释、可追溯、可分类的商品改进建议。

## 生成约束

每条建议至少要依赖以下三类证据中的两类：

- 评论依据
- 商品文案依据
- 商品图像依据

## 建议类型

对话中明确提出，应区分两类建议：

- 明确结构改进建议
- 用户感知驱动建议

## 为什么要区分

如果只有评论中提到问题，但商品文案和图像都没有清晰结构线索，那么系统不应过度声称“已经定位到具体结构缺陷”。

## 建议输出示例

```json
{
  "suggestion": "增强鞋面透气设计，优先改进前掌和足弓区域的通风性",
  "reason": [
    "多条评论提到夏季闷脚",
    "商品文案强调舒适，但缺少明确透气技术支撑",
    "现有图片显示鞋面包裹性较强，网孔结构不突出"
  ],
  "supporting_evidence": {
    "reviews": ["r_21", "r_56", "r_209"],
    "product_text_blocks": ["desc_3"],
    "product_images": ["img_4", "img_7"]
  },
  "confidence": 0.81
}
```
