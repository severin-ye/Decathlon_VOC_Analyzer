# 标识与对象关系

## 核心对象

系统至少包含以下对象：

- 商品
- 商品图片
- 商品文本块
- 评论
- 评论方面对象
- 召回记录
- 生成结果

## 推荐 ID 体系

- product_id：商品唯一标识
- image_id：商品图片唯一标识
- text_block_id：商品文案片段唯一标识
- review_id：评论唯一标识
- aspect_id：评论抽取后的方面级对象标识
- retrieval_id：一次召回动作的记录标识
- report_id：一次综合输出的结果标识

## 关系设计

- 一个 product_id 对应多个 image_id
- 一个 product_id 对应多个 text_block_id
- 一个 product_id 对应多个 review_id
- 一个 review_id 对应多个 aspect_id
- 一个 aspect_id 可以关联多个召回证据

## 为什么这一层要独立成文档

因为后续所有可解释性、聚合与评估，都依赖稳定的对象关系。如果这层不先定义清楚，召回记录、证据追踪和建议生成都会失去统一主键。
