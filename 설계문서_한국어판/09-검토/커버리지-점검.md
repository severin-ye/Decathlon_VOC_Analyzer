# 커버리지 점검

## 검토 목표

[대화 기록 원문](../../gpt%E8%81%8A%E5%A4%A9%E8%AE%B0%E5%BD%95.md) 와 대조하여, 현재 문서 시스템에 핵심 설계 정보 누락이 있는지 점검한다.

## 이미 커버된 핵심 정보

### 1. 멀티모달 RAG 기초

- 멀티모달 RAG 와 일반 RAG 의 차이를 이미 커버했다.
- Text-first, Dual-store, Native Multimodal 의 세 가지 큰 방향을 이미 커버했다.
- 이 부분이 요약만 남지 않도록, 독립적인 방안 비교 문서를 추가했다.

### 2. 원시 모달 리트리벌

- “원시 모달 보존”의 엔지니어링 주선을 이미 커버했다.
- page-level 에서 region-level 로의 진화 방향을 이미 커버했다.
- 2단계 리트리벌과 rerank 의 필요성을 이미 커버했다.
- 연구 및 엔지니어링 배경 맥락을 보존하기 위해 독립 진화 문서를 추가했다.

### 3. LangChain + Qwen 기술 경로

- LangChain, ChatQwen, Qwen3-VL-Embedding, Qwen3-VL-Reranker, Qdrant 의 역할 분담을 이미 커버했다.
- 먼저 API 생성, 점진적인 로컬 리트리벌과 재정렬의 제안을 이미 커버했다.
- 외부 링크, 내용 정리, 프로젝트 사용 방식에 따라 정리한 인용 하위 디렉터리를 추가했다.

### 4. 데카트론 상품 분석 시스템

- 상품 증거 패키지를 이미 커버했다.
- 리뷰 구조화 모델링을 이미 커버했다.
- 상품 인상과 고객 인상을 이미 커버했다.
- 종합 의견과 개선 제안 생성을 이미 커버했다.

### 5. 전 체인 설명 가능성

- 리트리벌 기록을 이미 커버했다.
- evidence span 을 이미 커버했다.
- 집계 통계를 이미 커버했다.
- 제안 출처 추적을 이미 커버했다.

### 6. 구현 단계

- MVP 를 이미 커버했다.
- Phase 2 를 이미 커버했다.
- Phase 3 를 이미 커버했다.

### 7. 논문 작성용 인용 준비

- [인용-개요](../08-참고-자료/인용/인용-개요.md) 를 추가했다.
- 외부 링크, 대응 내용 정리, 본 프로젝트 사용 부분, 사용 이유, 본 프로젝트 사용 아이디어에 따라 기술 출처를 정리했다.

## 현재 별도로 전개하지 않았지만 문서 안에 입구를 남겨 둔 정보

- 더 세밀한 region-level 시각 증거 정렬
- 더 정식화된 검색 평가 지표 체계
- 더 완전한 논문 출처 및 외부 링크 목록

이 내용들은 원본 대화에서 확장 방향에 속하며, 현재 프로젝트의 주선은 아니므로, 현재는 후속 확장 입구로만 남겨 두고 별도 장문의 부록으로 작성하지 않았다.

## 검토 결론

현재 문서 시스템은 원본 대화의 주선 설계 정보를 누락하지 않았다.

존재하는 것은 누락이 아니라 압축이며, 주로 두 가지다.

- 참고 자료는 모든 외부 링크를 항목별로 베껴 쓰지 않고 기술 방향별로 분류했다.
- 멀티모달 RAG 일반 지식 부분은 전체 개요가 아니라, 본 프로젝트와 관련된 필수 배경으로 압축했다.

이 두 가지 압축은 현재 문서 시스템의 “프로젝트 설계에 봉사한다”는 목표에 부합하며, 정보 결손을 구성하지 않는다.
