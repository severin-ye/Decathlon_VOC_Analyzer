# MVP 및 반복 계획

## MVP 단계

목표는 먼저 시스템의 폐루프를 끝까지 실행 가능하게 하는 것이지, 가장 복잡한 버전을 한 번에 완성하는 것이 아니다.

### MVP 범위

- 상품 이미지 + 상품 문안 통합 인덱스
- 리뷰에 대한 aspect 수준 구조화 추출
- 리뷰가 상품 증거 리트리벌을 구동
- product_id 기준 집계
- 종합 의견과 개선 제안 출력
- 전 과정에서 evidence id 유지

### MVP 에서 하지 않는 것

- 이미지 영역 수준 크롭
- 복잡한 고객 프로필 클러스터링
- 시간 추세 분석
- 다중 턴 agent 반성

## Phase 2

- 상품 인상 모델과 고객 인상 모델 추가
- 집계 통계와 집단 차이 분석 강화
- reranker 재정렬 도입

## Phase 3

- 이미지 영역 수준 증거 추가
- 리뷰와 이미지 국소 영역 정렬 도입
- 더 세밀한 개선 제안 생성

## 구현 순서의 핵심 이유

대화에서는 처음부터 가장 복잡한 region-level 원시 모달 검색을 수행하지 말고, 먼저 안정적인 상품 이미지 + 상품 문안 통합 리트리벌 폐루프를 구축해야 한다고 명확히 권장했다.
