# LangChain 과 Qwen

## 인용 1

### 외부 링크

https://docs.langchain.com/oss/python/integrations/chat/qwen

### 대응 내용 정리

LangChain 의 ChatQwen 통합 문서는 원본 대화에서 여러 핵심 역할을 담당했다.

- ChatQwen 이 이미지 입력과 비디오 입력을 지원한다는 점을 증명한다.
- ChatQwen 이 structured output 을 지원한다는 점을 증명한다.
- ChatQwen 이 tool calling 을 지원한다는 점을 증명한다.

### 본 프로젝트에서 사용하는 부분

- 리뷰 구조화 추출
- 최종 종합 의견 생성
- 개선 제안 구조화 출력

### 사용하는 이유

이 문서는 현재 프로젝트의 생성 계층과 오케스트레이션 계층에 대한 가장 직접적인 공식 근거다.

### 본 프로젝트에서 사용하는 아이디어

LangChain 으로 메시지와 흐름을 조직하고, ChatQwen 이 리뷰 구조화 출력과 최종 증거 제약 요약을 담당하게 하여, 생성 모델이 리트리벌 시스템을 대체하지 않도록 한다.

## 인용 2

### 외부 링크

https://docs.langchain.com/oss/python/langchain/messages

### 대응 내용 정리

이 문서는 LangChain 에서 멀티모달 메시지 콘텐츠 블록을 조직하는 방식을 설명한다.

### 본 프로젝트에서 사용하는 부분

- 상품 이미지 증거와 텍스트 증거의 메시지 패키징 방식

### 사용하는 이유

본 프로젝트는 최종적으로 상품 이미지, 상품 텍스트, 리뷰 증거를 함께 생성 모델에 전달해야 하므로, 안정적인 멀티모달 메시지 조직 방법이 필요하다.

### 본 프로젝트에서 사용하는 아이디어

후속 시스템 구현에서는 이 문서의 콘텐츠 블록 사고를 사용해, 이미지 증거, 텍스트 증거, 출처 식별자를 함께 생성 단계 입력으로 패키징한다.

## 인용 3

### 외부 링크

https://www.alibabacloud.com/help/en/model-studio/qwen-api-via-dashscope

### 대응 내용 정리

이 문서는 Qwen 멀티모달 생성 API 의 호출 입구와 연동 방식을 제공한다.

### 본 프로젝트에서 사용하는 부분

- 생성 모델의 API 연동 근거

### 사용하는 이유

원본 대화에서는 먼저 API 경로로 생성 계층을 끝까지 실행 가능하게 한 뒤, 리트리벌과 reranker 를 점진적으로 로컬의 제어 가능한 구성 요소로 만들어 가자고 명확히 제안했다.

### 본 프로젝트에서 사용하는 아이디어

생성 계층은 우선 DashScope 또는 Model Studio API 경로를 사용하여, 첫 버전의 폐루프 난도를 낮춘다.

## 인용 4

### 외부 링크

https://www.alibabacloud.com/help/en/model-studio/models

### 대응 내용 정리

이 문서는 Qwen 멀티모달 모델의 컨텍스트 길이, 단일 이미지 제한, 모델 간 차이 정보를 제공한다.

### 본 프로젝트에서 사용하는 부분

- 생성 모델 선택 근거
- 증거 수량 제어 근거

### 사용하는 이유

본 프로젝트는 여러 장의 상품 이미지와 몇 개의 텍스트 증거를 생성 모델에 전달해야 하므로, 입력 상한과 비용 경계를 명확히 해야 한다.

### 본 프로젝트에서 사용하는 아이디어

이를 증거 패키지 크기를 제약하는 데 사용한다. 예를 들어 최종 생성 시에는 모든 후보를 모델에 보내는 대신, 소수의 고품질 상품 이미지와 핵심 텍스트 블록만 전달한다.

## 인용 5

### 외부 링크

https://github.com/QwenLM/Qwen3-VL-Embedding

### 대응 내용 정리

Qwen3-VL-Embedding 공식 저장소는 원본 대화에서 검색과 재정렬 주선의 주요 근거를 담당했다.

- Embedding 모델이 initial recall 을 담당한다.
- Reranker 모델이 re-ranking stage 를 담당한다.
- 텍스트, 이미지, 스크린샷, 비디오, 혼합 입력을 지원한다.

### 본 프로젝트에서 사용하는 부분

- 상품 이미지와 상품 텍스트의 통합 벡터화
- 거친 리트리벌과 reranker 의 역할 구분
- 2B 와 8B 모델의 구현 계층 선택 논리

### 사용하는 이유

이것은 본 프로젝트의 Native Multimodal 리트리벌 경로에 대한 가장 직접적인 핵심 구성 요소 출처다.

### 본 프로젝트에서 사용하는 아이디어

MVP 에서는 더 가벼운 embedding 과 reranker 조합을 먼저 사용하여 상품 이미지와 문안 리트리벌, 리뷰 구동 질의를 먼저 연결하고, 이후 더 무거운 모델과 더 세밀한 시각 증거를 고려한다.
