# 문서 파싱과 구조화 리트리벌

## 인용 1

### 외부 링크

https://docs.langchain.com/oss/python/integrations/document_loaders/opendataloader_pdf

### 대응 내용 정리

이 문서는 문서 파싱 단계에서 읽기 순서, 표, 바운딩 박스 등의 구조 정보를 유지할 수 있으며, 모든 것을 단순히 평평한 순수 텍스트로 펼치지 않아도 된다는 점을 설명한다.

### 본 프로젝트에서 사용하는 부분

- 구조 충실성이 순수 텍스트화보다 우선된다는 사고

### 사용하는 이유

현재 프로젝트는 주로 상품 HTML 페이지이지 PDF 가 아니지만, 이 자료는 “구조 보존”의 중요성을 규정하는 데 도움을 준다.

### 본 프로젝트에서 사용하는 아이디어

이를 현재 프로젝트에 옮기면, 상품 증거 계층에서 이미지 유형, 문안 유형, 출처 관계를 유지하고, 라벨 없는 텍스트 한 단락만 남기지 않는다는 뜻이 된다.

## 인용 2

### 외부 링크

https://docs.langchain.com/oss/python/langchain/retrieval

### 대응 내용 정리

이 문서는 Hybrid RAG 가 보통 단일 검색 한 번으로 끝나지 않고, 질의 전처리, 검색, 검증, 후처리 등의 단계를 포함한다고 설명한다.

### 본 프로젝트에서 사용하는 부분

- 혼합 검색 관점
- 리트리벌 이후에도 후처리와 검증이 필요하다는 점

### 사용하는 이유

이 문서는 왜 본 프로젝트가 단일 단계 리트리벌이 아니라, 리트리벌, 병합, 재정렬, 증거 패키징을 수행하는지를 설명하는 프레임워크적 근거로 적합하다.

### 본 프로젝트에서 사용하는 아이디어

본 프로젝트의 이중 경로 리트리벌, product_id 병합, rerank, 증거 패키지 구성은 본질적으로 상품 분석 시나리오를 위한 Hybrid RAG 의 한 가지 구체화다.

## 인용 3

### 외부 링크

https://github.com/HKUDS/RAG-Anything/blob/main/README.md

### 대응 내용 정리

RAG-Anything 은 구조화 요소 보존과 관계 강화 멀티모달 검색 경로를 대표하며, 벡터 유사도와 그래프 탐색의 결합, 모달 인지 정렬, 관계 일관성을 강조한다.

### 본 프로젝트에서 사용하는 부분

- 구조화 멀티모달 리트리벌의 참고 경로
- 관계 강화 검색의 개념 배경

### 사용하는 이유

이것은 관련 작업에서 “모든 멀티모달 시스템이 순수 원시 모달 시각 검색 경로를 택하는 것은 아니다”라는 대조 경로로 적합하다.

### 본 프로젝트에서 사용하는 아이디어

현재 프로젝트는 아직 그래프 구조 리트리벌을 사용하지 않지만, 객체 관계 계층을 남겨 두어 향후 상품 이미지, 상품 텍스트, 리뷰 측면 객체, 제안 증거를 관계 그래프로 조직할 수 있는 공간을 확보했다.

## 인용 4

### 외부 링크

https://developer.nvidia.com/blog/how-to-build-a-document-processing-pipeline-for-rag-with-nemotron/

### 대응 내용 정리

이 문서는 더 엔지니어링 지향적인 구조화 멀티모달 리트리벌 사고를 보여준다.

- 표 markdown 보존
- 차트 크롭 보존
- 텍스트, 이미지 또는 이미지와 텍스트 결합 embedding 사용
- top-k 이후 multimodal reranking 수행

### 본 프로젝트에서 사용하는 부분

- “원시 요소 보존 + 구조 관계 보존 + 이후 멀티모달 재정렬”이라는 엔지니어링 사고

### 사용하는 이유

이는 본 프로젝트의 “모든 모달을 원문 그대로 모델에 넣는 것이 아니라, 핵심 증거만 최종 생성 단계에 투입한다”는 원칙을 뒷받침하기에 특히 적합하다.

### 본 프로젝트에서 사용하는 아이디어

현재 프로젝트에서 상품 이미지, 상품 문안, 리뷰 측면 객체 세 가지를 증거 패키지로 묶는 방식은 본질적으로 이러한 구조 보존 후 재정렬 사고와 일치한다.

## 인용 5

### 외부 링크

https://github.com/docling-project/docling

### 대응 내용 정리

Docling 은 PDF, HTML, 이미지 등 다양한 문서 구조 파싱 능력을 제공하며, layout, reading order, table structure, formula, lossless JSON 을 보존한다.

### 본 프로젝트에서 사용하는 부분

- 선택 가능한 문서 파싱 확장 도구

### 사용하는 이유

현재 프로젝트의 1단계는 PDF 를 주로 다루지 않지만, 향후 상품 설명서, 홍보물, 사양 부속 문서를 포함하게 되면 Docling 은 적절한 확장 항목이 된다.

### 본 프로젝트에서 사용하는 아이디어

현재 단계에서는 이를 기술 선택에 확장 옵션으로만 기록하고, MVP 의 핵심 의존성으로 두지 않는다.
