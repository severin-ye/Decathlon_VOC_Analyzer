# 후보였지만 채택하지 않은 경로

## 인용 1

### 외부 링크

https://github.com/openclaw/openclaw

### 대응 내용 정리

OpenClaw 는 개인 AI assistant 및 agent 플랫폼이며, 핵심은 채널 간 도우미, 도구 호출, 자동화에 있다. 전용 멀티모달 RAG 문서 검색 프레임워크는 아니다.

### 본 프로젝트에서 사용하는 부분

- 관련 작업에서의 비교 배경

### 사용하는 이유

원본 대화에서 OpenClaw 를 언급한 이유는, 그것이 원시 모달 리트리벌의 주선 방안으로 적합한지 판단하기 위해서였다.

### 본 프로젝트에서 사용하는 아이디어

본 프로젝트는 OpenClaw 를 핵심 아키텍처로 채택하지 않지만, 논문 관련 작업 또는 방안 비교에서 그 포지셔닝이 현재 프로젝트와 다르다는 점을 설명할 수 있다.

## 인용 2

### 외부 링크

https://github.com/openclaw/openclaw/issues/23200

### 대응 내용 정리

이 issue 는 핵심 판단 하나를 뒷받침하는 데 사용되었다. 즉, OpenClaw 는 현재 성숙한 native document indexing pipeline 을 갖추고 있지 않으며, 원시 모달 충실성을 지향하는 문서 리트리벌 주선을 형성하지도 못했다.

### 본 프로젝트에서 사용하는 부분

- “왜 OpenClaw 주선을 채택하지 않는가”에 대한 증거 출처

### 사용하는 이유

이는 논문이나 발표에서 “왜 더 agent 플랫폼처럼 보이는 어떤 방안을 선택하지 않았는가”를 설명하는 데 도움을 준다.

### 본 프로젝트에서 사용하는 아이디어

이를 역방향 논거로 사용해, 현재 프로젝트에 필요한 것은 범용 agent 플랫폼이 아니라 상품 증거 리트리벌과 리뷰 모델링 체인이라는 점을 설명한다.

## 인용 3

### 외부 링크

https://github.com/2dogsandanerd/ClawRag

### 대응 내용 정리

ClawRAG 는 현재 Docling + chunking + Vector/BM25/RRF 기반의 구조화 또는 혼합 검색 시스템에 더 가깝고, 멀티모달 검색은 여전히 후속 roadmap 에 속한다.

### 본 프로젝트에서 사용하는 부분

- 후보 경로 비교

### 사용하는 이유

이것은 RAG 라는 이름이 붙은 모든 프로젝트가 이미 원시 모달 리트리벌 경로까지 도달한 것은 아니라는 점을 보여준다.

### 본 프로젝트에서 사용하는 아이디어

현재 프로젝트는 이 경로를 핵심으로 채택하지 않지만, 관련 작업에서 텍스트 편향 및 혼합 검색의 대조군으로 사용할 수 있다.
