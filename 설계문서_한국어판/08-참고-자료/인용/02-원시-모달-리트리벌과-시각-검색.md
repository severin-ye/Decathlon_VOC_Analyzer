# 원시 모달 리트리벌과 시각 검색

## 인용 1

### 외부 링크

https://arxiv.org/abs/2407.01449

### 대응 내용 정리

이 자료는 원본 대화에서 ColPali 와 VisRAG 경로의 핵심 논거를 담당했다. 핵심 정보는 다음과 같다.

- 문서 페이지는 반드시 먼저 텍스트로 변환할 필요 없이, 직접 이미지 검색 객체가 될 수 있다.
- 하나의 페이지는 단일 벡터가 아니라 다중 벡터로 표현될 수 있다.
- query 와 페이지의 매칭은 late interaction 을 통해 수행되어, 더 세밀한 위치 지정 능력을 강화할 수 있다.

### 본 프로젝트에서 사용하는 부분

- 원시 모달 리트리벌의 이론적 핵심
- multi-vector 와 late interaction 의 배경
- page-level 에서 더 세밀한 위치 지정으로 나아가는 연구 기반

### 사용하는 이유

이 자료는 원시 모달 리트리벌 주선의 핵심 이론 근거이며, 논문 관련 작업과 방법 설계 배경에 두기 적합하다.

### 본 프로젝트에서 사용하는 아이디어

본 프로젝트는 상품 페이지를 PDF 페이지처럼 직접 page retrieval 하지는 않지만, 그 핵심 아이디어를 흡수한다. 즉, 상품 측에서 가능한 한 원시 시각 증거를 보존하고, 이후 국소 영역 증거 리트리벌을 위한 공간을 남겨 둔다.

## 인용 2

### 외부 링크

https://qdrant.tech/documentation/tutorials-search-engineering/pdf-retrieval-at-scale/

### 대응 내용 정리

Qdrant 튜토리얼은 다단계 검색 설계를 중점적으로 보여준다.

- 먼저 압축 표현 또는 pooled vector 로 거친 리트리벌을 수행한다.
- 이후 원시 multivector 로 정밀 재정렬을 수행한다.
- oversampling 을 통해 고품질 후보를 보존한다.

### 본 프로젝트에서 사용하는 부분

- 2단계 리트리벌 사고
- 거친 리트리벌과 재정렬의 분리
- 후보 과샘플링 감각

### 사용하는 이유

이 자료는 연구형 원시 모달 리트리벌을 생산 시스템에 더 가까운 엔지니어링 흐름으로 바꾸어 보여 주므로, 구현 계층의 방법 근거로 적합하다.

### 본 프로젝트에서 사용하는 아이디어

현재 프로젝트는 먼저 이중 경로 리트리벌과 reranker 로 폐루프를 연결하고, 이후 더 복잡한 시각 검색으로 업그레이드할 경우 이 2단계 프레임워크를 직접 참조할 수 있다.

## 인용 3

### 외부 링크

https://qdrant.tech/course/multi-vector-search/module-3/multi-stage-retrieval/

### 대응 내용 정리

이 자료는 다중 벡터 시나리오에서 다단계 리트리벌의 필요성을 더 설명하며, 먼저 후보 집합을 축소한 뒤 소규모 후보 집합에 대해 고비용 재정렬을 수행해야 한다고 강조한다.

### 본 프로젝트에서 사용하는 부분

- 다단계 리트리벌의 엔지니어링 필요성

### 사용하는 이유

논문 방법 장에서 “왜 리트리벌과 재정렬을 계층화해야 하는가”를 논증하는 데 적합하다.

### 본 프로젝트에서 사용하는 아이디어

본 프로젝트에서는 이것이 다음과 같이 구체화된다. 리뷰 측면 query 가 이미지 인덱스와 텍스트 인덱스의 초기 리트리벌을 구동한 뒤, reranker 를 통해 고품질 상품 증거 패키지로 집계된다.

## 인용 4

### 외부 링크

https://github.com/illuin-tech/colpali

### 대응 내용 정리

ColPali 공식 저장소의 핵심 기여는 원시 페이지 이미지의 다중 벡터 표현과 late interaction 검색 사고에 있다.

### 본 프로젝트에서 사용하는 부분

- 원시 모달 검색 패러다임 참고
- multi-vector 검색 개념의 출처

### 사용하는 이유

이것은 “연구 최전선이지만 현재 프로젝트가 아직 완전히 채택하지는 않은” 대표 경로로 적합하다.

### 본 프로젝트에서 사용하는 아이디어

현재 프로젝트는 ColPali 방안을 직접 재사용하지 않지만, 이후 상품 이미지의 국소 영역 수준 리트리벌에 대한 연구 참고로 삼는다.

## 인용 5

### 외부 링크

https://arxiv.org/html/2510.15253v2

### 대응 내용 정리

이 개요는 시각 문서 검색이 page-level 원자 단위에서 더 세밀한 페이지 내부 영역, 요소 수준, 계층식 인덱스로 진화하고 있음을 강조한다.

### 본 프로젝트에서 사용하는 부분

- page-level 에서 region-level 로의 진화 방향
- 계층식 증거 조직의 연구 배경

### 사용하는 이유

이것은 본 프로젝트가 왜 먼저 객체 수준 리트리벌을 수행하고, 이후 영역 수준 증거를 위한 인터페이스를 남겨 두는지를 뒷받침하기에 적합하다.

### 본 프로젝트에서 사용하는 아이디어

본 프로젝트의 MVP 는 여전히 상품 이미지 객체 수준 리트리벌을 중심으로 하지만, Phase 3 에서 신발 갑피, 밑창, 렌즈 등의 영역 수준 증거 확장을 남겨 둔다.

## 인용 6

### 외부 링크

https://arxiv.org/abs/2510.27261

### 대응 내용 정리

RegionRAG 경로는 전체 페이지 노이즈가 생성 단계에 미치는 간섭을 줄이기 위해, 직접 region-level retrieval 을 수행하는 것을 강조한다.

### 본 프로젝트에서 사용하는 부분

- 영역 수준 시각 증거 리트리벌의 필요성

### 사용하는 이유

이는 이후 “상품 국소 구조도와 리뷰 측면을 정렬한다”는 연구 방향을 뒷받침하는 근거가 될 수 있다.

### 본 프로젝트에서 사용하는 아이디어

본 프로젝트는 현재 RegionRAG 를 직접 도입하지 않지만, 향후 신발 갑피 통기 영역, 밑창 쿠셔닝 구조 영역의 증거 리트리벌을 수행하려 할 때 직접 참고할 수 있다.

## 인용 7

### 외부 링크

https://arxiv.org/html/2508.00579v2

### 대응 내용 정리

MMRAG-DocQA 는 계층식 인덱스 경로를 대표하며, 페이지 내부 블록, 페이지 간 위상 관계, 상위 페이지 리트리벌, 문서 요약 검색이 함께 작동할 수 있음을 강조한다.

### 본 프로젝트에서 사용하는 부분

- 계층식 증거 조직 사고

### 사용하는 이유

이것은 본 프로젝트가 미래에 “단일 상품 증거 객체”에서 “상품 증거 + 구조 관계” 경로로 확장되는 것을 뒷받침하기에 적합하다.

### 본 프로젝트에서 사용하는 아이디어

현재 프로젝트는 객체 계층에서 product_id, image_id, text_block_id, review_id 의 관계를 이미 구축했다. 향후 더 강한 구조 관계를 추가한다면, 이 자료를 이론 참고로 사용할 수 있다.

## 인용 8

### 외부 링크

https://arxiv.org/abs/2410.10594

### 대응 내용 정리

본 프로젝트에서 이는 Phase 3 의 상품 이미지 영역 크롭 및 국소 증거 리트리벌에 해당한다.
