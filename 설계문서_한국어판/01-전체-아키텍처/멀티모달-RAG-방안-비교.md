# 멀티모달 RAG 방안 비교

## 목적

이 문서는 원본 대화에서 멀티모달 RAG 의 세 가지 주류 구현 방안에 대해 비교한 내용을 별도로 정리하여, 이러한 배경 판단이 프로젝트 문서에서 압축되며 사라지지 않도록 하기 위한 것입니다.

## 방안 A: Text-first

### 핵심 아이디어

먼저 이미지, 표, 차트, 비디오 등의 내용을 가능한 한 텍스트로 변환한 뒤, 전통적인 텍스트 RAG 를 적용한다.

### 전형적 방법

- 이미지를 OCR + caption 으로 변환
- 표를 Markdown 또는 HTML 로 변환
- 차트를 설명 텍스트로 변환
- 비디오를 ASR 과 키프레임 설명으로 변환

### 적합한 시나리오

- 빠른 프로토타입
- 제한된 비용
- 시각 세부 요소 요구가 극단적으로 높지 않은 경우

### 한계

- 시각 세부 요소 손실이 뚜렷하다.
- 이미지와 차트의 의미는 전사 품질에 크게 의존한다.

## 방안 B: Dual-store

### 핵심 아이디어

원시 모달과 텍스트화된 요약을 동시에 보존하고, 검색 및 생성 단계에서 필요에 따라 조합해 사용한다.

### 전형적 방법

- 이미지는 원본 이미지를 유지하면서 caption 또는 OCR 요약을 함께 저장
- 표는 구조화 표현을 유지하면서 텍스트 전사를 함께 저장
- 검색 단계는 이중 경로 리트리벌을 수행
- 생성 단계는 핵심 원시 증거만 모델에 전달

### 적합한 시나리오

- 엔터프라이즈 문서 질의응답
- 상품 이미지와 문안 분석
- 의료 영상과 보고서의 결합 분석

### 한계

- 설계 복잡도가 Text-first 보다 높다.
- 모달 간 정렬과 병합을 처리해야 한다.

## 방안 C: Native Multimodal

### 핵심 아이디어

- 검색 시점부터 멀티모달 embedding 을 사용한다.
- 생성 시 직접 멀티모달 대형 모델을 사용한다.
- 전 과정에서 text, image, video, audio 등의 멀티모달 입력을 지원한다.

### 적합한 시나리오

- 고예산 프로젝트
- 연구형 프로젝트
- 차트와 시각 증거 요구가 높은 작업

### 한계

- 엔지니어링이 복잡하다.
- 비용이 높다.
- 튜닝이 어렵다.

## 현재 프로젝트에 대한 결론

대화에서의 최종 판단은 “순수한 Native Multimodal 을 전면 채택”하는 것이 아니라, 다음과 같습니다.

- 상품 증거 리트리벌에는 Native Multimodal 성향을 채택한다.
- 리뷰 이해에는 구조화된 텍스트 모델링을 채택한다.
- 최종 생성에는 증거 제약을 받는 멀티모달 요약을 채택한다.

따라서 현재 프로젝트를 더 정확하게 말하면 “Native Multimodal 을 리트리벌 핵심으로 하는 혼합 멀티모달 시스템”입니다.
