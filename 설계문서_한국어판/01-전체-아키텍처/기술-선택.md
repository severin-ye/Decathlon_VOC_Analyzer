# 기술 선택

## 목표 선택안

대화에서 최종적으로 수렴된 추천 조합은 다음과 같습니다.

- 오케스트레이션 계층: LangChain
- 생성 계층: ChatQwen + Qwen VL 시리즈
- 리트리벌 계층: Qwen3-VL-Embedding
- 재정렬 계층: Qwen3-VL-Reranker
- 벡터 데이터베이스: Qdrant
- 문서 파싱 확장: Docling

## 역할 분담

### LangChain

- 전체 체인을 오케스트레이션한다.
- 멀티모달 메시지를 조직한다.
- structured output 을 처리한다.
- 도구 호출을 이어받는다.

### Qwen VL 생성 모델

- 최종 증거 패키지를 읽는다.
- 종합 의견을 출력한다.
- 개선 제안을 출력한다.
- 구조화 결과를 출력한다.

### Qwen3-VL-Embedding

- 상품 이미지와 상품 텍스트를 통합 인코딩한다.
- query 와 상품 증거를 위한 공유 의미 공간을 구축한다.
- 초기 리트리벌을 지원한다.

### Qwen3-VL-Reranker

- 상품 이미지와 상품 텍스트의 후보 증거를 재정렬한다.
- embedding 기반 거친 리트리벌이 가져오는 노이즈를 줄인다.

### Qdrant

- 이미지와 텍스트 인덱스를 저장한다.
- 이후 멀티벡터 또는 더 복잡한 schema 확장을 지원한다.
- 생산급 리트리벌과 평가에 적합하다.

## 왜 순수 OpenClaw 경로를 선택하지 않는가

대화에서의 판단은 다음과 같습니다.

- OpenClaw 본체는 멀티모달 RAG 프레임워크의 핵심이 아니다.
- ClawRAG 는 현재 Docling + 텍스트 또는 혼합 검색에 더 치우쳐 있다.
- 이것은 현재 프로젝트가 필요로 하는 원시 모달 리트리벌의 주선이 아니다.

## 모델 사용 제안

- 프로토타입 우선: 더 가벼운 embedding + API 생성 모델
- 연구 지향: 로컬 embedding 과 reranker, 생성 모델은 API 가능
- 고품질 단계: region-level 증거와 더 강한 reranker 도입
