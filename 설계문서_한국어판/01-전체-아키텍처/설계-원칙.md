# 설계 원칙

## 원칙 1: 단일 책임

각 모듈은 한 종류의 문제만 처리합니다.

- 데이터 계층은 모델 추론을 담당하지 않는다.
- 리트리벌 계층은 직접 개선 제안을 출력하지 않는다.
- 리뷰 모델링 계층은 직접 최종 요약을 하지 않는다.
- 생성 계층은 리트리벌 계층과 통계 계층을 대체하지 않는다.

## 원칙 2: 단계별 충실성 유지

- 상품 이미지와 상품 문안은 가능한 한 원시 의미를 유지한다.
- 리뷰 내용은 가능한 한 구조화하여 이후의 블랙박스 추론을 줄인다.
- 최종 생성 단계는 선별된 증거 패키지만 소비한다.

## 원칙 3: 결론보다 증거 우선

- 먼저 증거가 있고, 그다음 인상이 있으며, 그다음 결론이 있고, 그다음 제안이 있다.
- 모든 최종 출력은 반드시 증거 ID까지 추적 가능해야 한다.

## 원칙 4: 순수 시각 엔드투엔드가 아니라 혼합 멀티모달

- 상품 측은 멀티모달 리트리벌을 사용한다.
- 리뷰 측은 텍스트 구조화 모델링을 사용한다.
- 집계 계층은 통계와 규칙 제약을 사용한다.
- 최종 생성 단계에서 다시 멀티모달 대형 모델을 호출한다.

## 원칙 5: 반복적 구현

먼저 폐루프가 가능한 MVP를 만들고, 그다음 reranker, 영역 수준 리트리벌, 고객 인상 클러스터링, 세밀한 제안을 추가한다.
