# 시스템 총람

## 시스템 목표

시스템은 하나의 핵심 작업을 중심으로 전개됩니다.

- 상품 멀티모달 증거를 리트리벌한다.
- 리뷰에서 측면 수준 증거를 추출한다.
- 이를 상품 인상과 고객 인상으로 집계한다.
- 종합 의견과 개선 제안을 생성한다.

## 전체 계층 구조

시스템은 책임에 따라 여섯 개의 계층으로 나뉩니다.

1. 데이터 계층
2. 리트리벌 계층
3. 리뷰 모델링 계층
4. 집계 계층
5. 생성 및 제안 계층
6. 설명 가능성 및 평가 계층

## 데이터 흐름의 주선

입력에서 출력까지의 주 체인은 다음과 같습니다.

1. 상품 이미지, 상품 문안, 리뷰를 수집한다.
2. product_id 를 중심으로 상품 증거 패키지를 조직한다.
3. 상품 이미지와 상품 텍스트에 대해 각각 리트리벌 인덱스를 구축한다.
4. 리뷰에 대해 측면 수준 구조화 추출을 수행한다.
5. 리뷰에서 추출한 수요 또는 관점을 이용해 상품 증거 리트리벌을 구동한다.
6. 상품 증거와 리뷰 증거를 상품별로 집계한다.
7. 상품 인상, 고객 인상, 종합 의견, 개선 제안을 출력한다.

## 핵심 시스템 판단

대화에서 형성된 최종 아키텍처 판단은 다음과 같습니다.

- 상품 측은 멀티모달 리트리벌에 적합하다.
- 리뷰 측은 구조화된 텍스트 모델링에 적합하다.
- 생성 단계는 반드시 증거의 제약을 받아야 한다.
- 설명 가능성은 리트리벌, 추출, 집계, 생성의 네 단계를 관통해야 한다.

## 왜 순수 RAG 질의응답이 아닌가

프로젝트의 최종 출력은 한 줄의 답이 아니라, 상품 인사이트 결과 전체이기 때문입니다.

- 상품 장점
- 상품 단점
- 사용 장면 인상
- 고객 관심 포인트
- 논쟁 지점
- 설명 가능한 개선 제안
