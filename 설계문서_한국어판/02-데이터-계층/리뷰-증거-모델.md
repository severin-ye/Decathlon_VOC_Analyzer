# 리뷰 증거 모델

## 리뷰 계층은 순수 원시 텍스트로 직접 보존하지 않는다

대화에서의 명확한 판단은 다음과 같습니다. 리뷰 측의 주체는 여전히 텍스트이므로, 전체 리뷰를 원문 그대로 시각 모델에 던져 블랙박스 요약을 시키기보다는, 가능한 빨리 구조화된 증거로 전환하는 것이 가장 적합합니다.

## 리뷰 기본 필드

- review_id
- product_id
- rating
- review_text
- timestamp
- optional variant 정보

## 측면 수준 구조 객체

각 리뷰는 여러 개의 측면 객체로 분해되어야 하며, 각 측면 객체는 최소한 다음을 포함해야 합니다.

- aspect
- sentiment
- opinion
- evidence_span
- usage_scene
- confidence

## 구조화 예시

```json
{
  "review_id": "r_1024",
  "product_id": "p_1001",
  "aspects": [
    {
      "aspect": "쿠셔닝",
      "sentiment": "positive",
      "opinion": "착지감이 비교적 부드러워 오래 걸어도 피곤하지 않다",
      "evidence_span": "밑창이 매우 부드러워 하루 종일 걸어도 피곤하지 않다",
      "usage_scene": "장시간 보행/일상 통근",
      "confidence": 0.89
    },
    {
      "aspect": "통기성",
      "sentiment": "negative",
      "opinion": "여름에 신으면 답답하다",
      "evidence_span": "날씨가 더울 때는 발이 조금 답답하다",
      "usage_scene": "여름철",
      "confidence": 0.84
    }
  ],
  "overall_sentiment": "mixed"
}
```

## 이 계층의 책임

- 리트리벌 계층에 더 안정적인 query 출처를 제공한다.
- 집계 계층에 통계 가능한 구조화 객체를 제공한다.
- 생성 계층에 인용 가능한 원문 문장 증거를 제공한다.
