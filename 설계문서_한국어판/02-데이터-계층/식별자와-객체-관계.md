# 식별자와 객체 관계

## 핵심 객체

시스템은 최소한 다음 객체들을 포함합니다.

- 상품
- 상품 이미지
- 상품 텍스트 블록
- 리뷰
- 리뷰 측면 객체
- 리트리벌 기록
- 생성 결과

## 권장 ID 체계

- product_id: 상품 고유 식별자
- image_id: 상품 이미지 고유 식별자
- text_block_id: 상품 문안 조각 고유 식별자
- review_id: 리뷰 고유 식별자
- aspect_id: 리뷰 추출 후의 측면 수준 객체 식별자
- retrieval_id: 한 번의 리트리벌 동작에 대한 기록 식별자
- report_id: 한 번의 종합 출력 결과 식별자

## 관계 설계

- 하나의 product_id 는 여러 개의 image_id 에 대응한다.
- 하나의 product_id 는 여러 개의 text_block_id 에 대응한다.
- 하나의 product_id 는 여러 개의 review_id 에 대응한다.
- 하나의 review_id 는 여러 개의 aspect_id 에 대응한다.
- 하나의 aspect_id 는 여러 개의 리트리벌 증거와 연관될 수 있다.

## 왜 이 계층을 별도 문서로 분리해야 하는가

이후의 모든 설명 가능성, 집계, 평가가 안정적인 객체 관계에 의존하기 때문입니다. 이 계층이 먼저 명확히 정의되지 않으면, 리트리벌 기록, 증거 추적, 제안 생성은 모두 통일된 기본 키를 잃게 됩니다.
