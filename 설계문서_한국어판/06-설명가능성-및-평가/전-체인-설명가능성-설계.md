# 전 체인 설명 가능성 설계

## 설계 목표

프로젝트는 각 단계에서 가능한 한 설명 가능성을 유지하도록 요구하므로, 설명 가능성은 부가 기능이 아니라 시스템 설계 제약이어야 한다.

## 리트리벌 단계의 설명 가능성

각 리트리벌은 다음을 기록해야 한다.

- query 내용
- source review_id
- source aspect
- 적중한 image_id 또는 text_block_id
- embedding_score
- rerank_score

## 리뷰 추출 단계의 설명 가능성

각 측면 객체는 반드시 다음을 보존해야 한다.

- evidence_span
- usage_scene
- confidence

## 집계 단계의 설명 가능성

종합 의견 생성 전에 먼저 다음과 같은 중간 통계 객체를 산출해야 한다.

- aspect_frequency
- positive_ratio
- negative_ratio
- scene_distribution
- representative_reviews

## 제안 단계의 설명 가능성

각 제안은 반드시 다음으로 역참조될 수 있어야 한다.

- 어떤 리뷰들인지
- 어떤 상품 문안들인지
- 어떤 상품 이미지들인지

## 결과의 용도

이 메커니즘은 다음을 지원하는 데 사용된다.

- 결론이 타당한지 검토한다.
- 리트리벌이 틀렸는지 요약이 틀렸는지 분석한다.
- 논문 속 error analysis 를 수행한다.
- 발표 또는 비즈니스 측에 증거 사슬을 제시한다.
