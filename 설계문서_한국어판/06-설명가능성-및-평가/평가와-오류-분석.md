# 평가와 오류 분석

## 평가 목표

평가는 최종 생성의 품질만 보면 안 되고, 계층별로 평가해야 한다.

- 리트리벌이 맞는지
- 리뷰 추출이 안정적인지
- 집계가 정확한지
- 제안이 증거의 뒷받침을 받는지

## 권장 계층별 평가

### 리트리벌 계층

- 올바른 상품 이미지를 리트리벌할 수 있는지
- 올바른 상품 문안을 리트리벌할 수 있는지
- reranker 가 단순 top-k embedding 보다 분명히 우수한지

### 리뷰 모델링 계층

- aspect 추출 정확성
- sentiment 방향이 안정적인지
- evidence_span 이 원문 문장과 일치하는지
- usage_scene 이 유효한지

### 생성 계층

- 종합 의견이 주요 장단점을 포괄하는지
- 개선 제안에 과도한 추론이 존재하는지
- 생성 결과가 실제 증거를 인용했는지

## 오류 위치 지정 사고방식

결과가 틀렸을 때 우선 다음을 판단한다.

1. 리트리벌이 틀렸다.
2. 측면 추출이 틀렸다.
3. 집계 통계가 틀렸다.
4. 생성 시 과도하게 확장했다.

## 미래 평가 확장

대화에서 언급된 Qdrant 다단계 검색 평가 방향은 후기 단계에서 더 정식적인 Recall@k, MRR, NDCG 지표 체계로 확장하는 데 활용할 수 있다.
