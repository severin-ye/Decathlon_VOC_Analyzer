# 종합 의견 생성

## 입력 제약

종합 의견은 원시 전체 데이터를 직접 기반으로 생성해서는 안 되며, 이미 조직된 증거 패키지를 기반으로 해야 한다.

- 상품 이미지 증거
- 상품 문안 증거
- 리뷰 측면 수준 증거
- 집계 통계 결과
- 대표 리뷰

## 출력 목표

종합 의견은 다음 내용을 포괄해야 한다.

- 상품 전체 평가
- 주요 장점
- 주요 단점
- 논쟁 지점
- 적용 장면 또는 적용 인구

## 생성 원칙

- 먼저 통계하고, 그다음 요약한다.
- 먼저 증거를 선별하고, 그다음 생성한다.
- 결론은 반드시 최소한 한 종류의 추적 가능한 증거에 대응해야 한다.

## 왜 직접 자유 생성하면 안 되는가

증거 제약이 없으면 시스템은 두 가지 능력을 잃게 된다.

- 결과 복기 능력
- 오류 위치 지정 능력

## 권장 출력 형식

구조화 출력을 권장한다.

- answer
- supporting_aspects
- supporting_reviews
- supporting_product_evidence
- confidence
