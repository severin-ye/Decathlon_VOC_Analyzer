# 개선 제안 생성

## 본 계층의 목표

종합 의견을 바탕으로, 설명 가능하고 추적 가능하며 분류 가능한 상품 개선 제안을 출력한다.

## 생성 제약

각 제안은 최소한 다음 세 종류 증거 중 두 종류에 의존해야 한다.

- 리뷰 근거
- 상품 문안 근거
- 상품 이미지 근거

## 제안 유형

대화에서는 두 종류의 제안을 구분해야 한다고 명확히 제시했다.

- 명확한 구조 개선 제안
- 사용자 인지 기반 제안

## 왜 구분해야 하는가

리뷰에만 문제가 언급되고, 상품 문안과 이미지에는 모두 뚜렷한 구조 단서가 없다면, 시스템은 “이미 구체적인 구조 결함까지 위치를 특정했다”고 과도하게 주장해서는 안 된다.

## 제안 출력 예시

```json
{
  "suggestion": "갑피의 통기 설계를 강화하고, 전족부와 아치 구역의 환기성을 우선 개선한다",
  "reason": [
    "여러 리뷰에서 여름철 발 답답함을 언급했다",
    "상품 문안은 편안함을 강조하지만, 명확한 통기 기술 근거가 부족하다",
    "현재 이미지는 갑피의 밀착성이 강하고 망사 구조가 두드러지지 않음을 보여준다"
  ],
  "supporting_evidence": {
    "reviews": ["r_21", "r_56", "r_209"],
    "product_text_blocks": ["desc_3"],
    "product_images": ["img_4", "img_7"]
  },
  "confidence": 0.81
}
```
