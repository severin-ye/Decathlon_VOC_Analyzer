# 이중 경로 리트리벌과 재정렬

## 이중 경로 리트리벌

시스템은 동시에 다음을 수행합니다.

- 이미지 리트리벌
- 텍스트 리트리벌

이후 리트리벌 결과는 단일 증거 수준에 머무르지 않고 product_id 기준으로 병합됩니다.

## 재정렬의 필요성

대화에서 이미 분명히 지적되었듯이, 멀티모달 리트리벌은 순수 텍스트 리트리벌보다 노이즈를 더 쉽게 유입시키므로, reranker 는 강력히 권장되는 구성 요소입니다.

## 병합 로직

병합된 증거 기록은 최소한 다음을 포함해야 합니다.

- product_id
- image_id 또는 text_block_id
- source_review_id
- source_aspect
- embedding_score
- rerank_score

## 예시 기록

```json
{
  "query": "여름에 발이 답답함",
  "source_review_id": "r_1024",
  "retrieved": [
    {
      "product_id": "p_1001",
      "image_id": "img_4",
      "text_id": "desc_3",
      "embedding_score": 0.79,
      "rerank_score": 0.91
    }
  ]
}
```

## 중장기 진화

대화에서는 미래 확장을 위한 더 선진적인 리트리벌 방식도 논의되었습니다.

- 2단계 거친 리트리벌 + 재정렬
- page-level 에서 region-level 로
- multi-vector late interaction
- 더 세밀한 evidence packing

이들은 현재 프로젝트의 후기 단계에서 도입할 수 있는 역량이며, MVP 의 출발점은 아닙니다.
