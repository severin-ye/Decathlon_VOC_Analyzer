# 원시 모달 리트리벌의 진화

## 목적

이 문서는 원본 대화에서 “원시 모달 리트리벌 유지”에 관한 엔지니어링 및 연구 진화 맥락을 별도로 보존하기 위한 것으로, 이 부분이 프로젝트 수준 요약만 남지 않도록 한다.

## 1단계: 전체 페이지를 이미지 리트리벌 대상으로 사용

이것은 ColPali, ColQwen, VisRAG 류 방법의 전형적인 출발점입니다.

### 핵심 특징

- 페이지를 직접 이미지 객체로 간주한다.
- 텍스트 query 와 페이지 이미지가 함께 공유 의미 공간에 들어간다.
- 먼저 페이지를 완전히 텍스트로 파싱하는 것에 의존하지 않는다.

### 주요 가치

- 파싱 단계의 정보 손실을 가능한 한 줄인다.
- 차트, 레이아웃, 국소 시각 증거에 더 우호적이다.

## 2단계: 2단계 리트리벌

대화에서는 엔지니어링 실무상, 무거운 원시 모달 표현으로 전체 정밀 검색을 직접 수행하지 않고 보통 2단계 흐름을 사용한다고 명확히 지적했습니다.

### Stage A: 거친 리트리벌

- pooled vector
- single-vector prefetch
- 희소 검색 또는 OCR 텍스트 보조 리트리벌

### Stage B: 재정렬

- multi-vector
- late interaction
- multimodal reranker

### 핵심 이유

- 계산 비용을 낮춘다.
- 후보 커버리지를 높인다.
- 실제로 관련 있는 객체가 재정렬 단계에 들어가도록 한다.

## 3단계: page-level 에서 region-level 로

최근의 추세는 리트리벌 단위를 전체 페이지에서 페이지 내부 영역으로 더 세분화하는 것입니다.

### 방향 A: 먼저 페이지를 리트리벌한 뒤 영역을 위치 지정

- 먼저 관련 페이지를 확정한다.
- 그다음 차트, 표, 그림 설명, 국소 구조 블록을 위치 지정한다.

### 방향 B: 직접 영역 수준 리트리벌

- 국소 영역을 직접 검색 가능한 객체로 본다.
- 전체 페이지 노이즈가 생성 단계에 미치는 간섭을 줄인다.

### 방향 C: 계층식 인덱스

- 페이지 수준, 영역 수준, 페이지 간 관계, 요약 계층을 동시에 유지한다.
- 서로 다른 입도 증거가 함께 리트리벌에 참여하도록 한다.

## 현재 프로젝트에 대한 시사점

현재 데카트론 상품 분석 프로젝트에서 가장 합리적인 구현 순서는 다음과 같습니다.

1. 먼저 상품 이미지와 상품 문안의 객체 수준 통합 리트리벌을 수행한다.
2. 그다음 이중 경로 병합과 reranker 를 수행한다.
3. 이후 상품 이미지의 국소 영역 수준 증거로 확장한다.

## 왜 이 문서가 별도로 존재하는가

이 부분의 내용은 “검색 연구 및 엔지니어링 진화 배경”에 속하며, 프로젝트 설계를 뒷받침해야 하지만 구체적인 비즈니스 프로세스 문서에 섞여서는 안 되기 때문입니다.
