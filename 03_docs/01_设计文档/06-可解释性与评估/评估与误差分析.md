# 评估与误差分析

## 评估目标

评估不能只看最终生成好不好，还要分层评估：

- 召回是否对
- 评论抽取是否稳
- 聚合是否准确
- 建议是否有证据支撑

## 建议分层评估

### 召回层

- 是否能召回正确商品图像
- 是否能召回正确商品文案
- reranker 是否明显优于单纯 top-k embedding

### 评论建模层

- aspect 抽取准确性
- sentiment 方向是否稳定
- evidence_span 是否与原句一致
- usage_scene 是否可用

### 生成层

- 综合意见是否覆盖主要优缺点
- 改进建议是否存在过度推断
- 生成结果是否引用了真实证据

## 误差定位思路

当结果错误时，优先判断：

1. 召回错了
2. 方面抽取错了
3. 聚合统计错了
4. 生成时过度发挥了

## 未来评估扩展

对话中提到的 Qdrant 多阶段检索评估方向，可用于后期扩展更正式的 Recall@k、MRR、NDCG 指标体系。
