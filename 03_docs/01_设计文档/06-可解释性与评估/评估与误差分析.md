# 评估与误差分析

## 评估目标

评估不能只看最终生成好不好，还要分层评估：

- 召回是否对
- 评论抽取是否稳
- 聚合是否准确
- 建议是否有证据支撑

## 建议分层评估

### 召回层

- 是否能召回正确商品图像
- 是否能召回正确商品文案
- reranker 是否明显优于单纯 top-k embedding

### 评论建模层

- aspect 抽取准确性
- sentiment 方向是否稳定
- evidence_span 是否与原句一致
- usage_scene 是否可用

### 生成层

- 综合意见是否覆盖主要优缺点
- 改进建议是否存在过度推断
- 生成结果是否引用了真实证据

## 误差定位思路

当结果错误时，优先判断：

1. 召回错了
2. 方面抽取错了
3. 聚合统计错了
4. 生成时过度发挥了

## 当前已落地的正式检索评测能力

- manifest evaluator 已支持可选 gold labels
- 标注可从 `manifest.evaluation_labels.retrieval_relevance` 或 `analysis.retrieval_labels` 读取
- 在有标注时可计算 `Recall@1/3/5`、`MRR`、`NDCG@3/5`
- 在无标注时继续输出当前 proxy metrics，兼容已有运行产物

## 后续评估扩展

- 建立覆盖多品类的冻结 retrieval 标注集
- 补上“评论直召回”与“问题驱动召回”的统一离线对照
- 在正式实验阶段输出更稳定的分组统计与误差分桶分析
