# 原始模态召回与视觉检索

## 引用 1

### 外链

https://arxiv.org/abs/2407.01449

### 对应内容整理

这条资料在原对话中承担了 ColPali 和 VisRAG 路线的核心论据。关键信息包括：

- 文档页可直接作为图像检索对象，而非必须先转文本。
- 一个页面可以表示为多向量，而不是一个单向量。
- query 和页面的匹配可通过 late interaction 完成，以增强细粒度定位能力。

### 本项目使用的部分

- 原始模态召回的理论核心
- multi-vector 与 late interaction 的背景
- 从 page-level 走向更细粒度定位的研究基础

### 使用的原因

这条资料是原始模态召回主线的核心理论依据，适合放在论文相关工作与方法设计背景中。

### 本项目使用的思路

本项目不直接把商品页当 PDF 页来做 page retrieval，但吸收其关键思想：在商品侧尽量保留原始视觉证据，并为后续局部区域证据召回预留空间。

## 引用 2

### 外链

https://qdrant.tech/documentation/tutorials-search-engineering/pdf-retrieval-at-scale/

### 对应内容整理

Qdrant 教程重点展示了多阶段检索设计：

- 先用压缩表示或 pooled vector 粗召回
- 再用原始 multivector 精排
- 通过 oversampling 保留高质量候选

### 本项目使用的部分

- 两阶段召回思路
- 粗召回与精排分离
- 候选过采样意识

### 使用的原因

它将研究型原始模态召回转换成更接近生产系统的工程流程，适合作为实现层方法依据。

### 本项目使用的思路

当前项目先用双路召回加 reranker 打通闭环，未来若升级到更复杂视觉检索，可直接借鉴这种两阶段框架。

在本项目里，它被具体化为：评论方面 query 驱动图像索引与文本索引初步召回，再通过 reranker 聚合成高质量商品证据包。
