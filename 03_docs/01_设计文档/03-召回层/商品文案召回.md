# 商品文案召回

## 召回对象

商品文案召回覆盖：

- 标题
- 卖点 bullet points
- 长描述
- 参数
- 规格说明

## 建议模型

使用同一套多模态 embedding 模型编码文本，以保证评论 query 与商品文本能够进入同一语义空间。

## 在线查询逻辑

1. 从评论方面对象生成 query。
2. 在商品文案索引中召回相关 text_block_id。
3. 返回文本候选及 embedding score。

## 这一层解决的问题

- 给评论观点找到商品官方解释或设计意图。
- 支持后续区分“用户感知问题”与“文案/结构已有明确描述但用户不认可”的情况。

## 为什么不能只依赖这一层

因为很多商品特征具有明显视觉属性，仅靠文字描述无法充分解释用户感知。
