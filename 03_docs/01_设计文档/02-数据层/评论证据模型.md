# 评论证据模型

## 评论层不直接保留为纯原始文本

对话中的明确判断是：评论侧主体仍然是文本，因此最适合尽快转成结构化证据，而不是把全部评论原样扔给视觉模型做黑箱总结。

## 评论基础字段

- review_id
- product_id
- rating
- review_text
- timestamp
- optional variant 信息

## 方面级结构对象

每条评论要被拆成多个方面对象，每个方面对象至少包含：

- aspect
- sentiment
- opinion
- evidence_span
- usage_scene
- confidence

## 结构化示例

```json
{
  "review_id": "r_1024",
  "product_id": "p_1001",
  "aspects": [
    {
      "aspect": "缓震",
      "sentiment": "positive",
      "opinion": "落地比较软，长时间走路不累",
      "evidence_span": "鞋底很软，走一天也不累",
      "usage_scene": "长时间步行/日常通勤",
      "confidence": 0.89
    },
    {
      "aspect": "透气性",
      "sentiment": "negative",
      "opinion": "夏天穿会闷",
      "evidence_span": "天气热的时候有点闷脚",
      "usage_scene": "夏季",
      "confidence": 0.84
    }
  ],
  "overall_sentiment": "mixed"
}
```

## 这一层的职责

- 提供给召回层更稳定的 query 来源
- 提供给聚合层可统计的结构化对象
- 提供给生成层可引用的原句证据
