# 消融实验与对照实验运行指南

## 文件结构

- `05_src/decathlon_voc_analyzer/workflows/experiment_runner.py` — 实验矩阵运行器
- `05_src/decathlon_voc_analyzer/workflows/llm_judge_evaluation.py` — LLM-as-Judge 评估器
- `05_src/decathlon_voc_analyzer/workflows/control_experiments.py` — 对照实验基线实现

## 实验条件（8个）

1. `full_system` — 完整系统（基线）
2. `ablation_no_qp` — 消融：无 Question Planning
3. `ablation_no_image` — 消融：无 Image Route
4. `ablation_no_rerank` — 消融：无 Reranking
5. `ablation_no_attribution` — 消融：无 Claim Attribution
6. `control_lewis2020` — 对照：Lewis et al. 2020（标准 RAG）
7. `control_jarvis` — 对照：JARVIS（证据图 + LLM 裁决）
8. `control_vericite` — 对照：VeriCite（三阶段引用验证）

## 运行实验矩阵

```bash
cd /home/severin/Codelib/Decathlon_VOC_Analyzer
.venv/bin/python -m decathlon_voc_analyzer.workflows.experiment_runner \
    --categories backpack shoes sunglasses \
    --products-per-category 5 \
    --max-reviews 25 \
    --output-dir ./experiment_results
```

### 参数说明

- `--categories`: 品类列表，默认 `["backpack", "shoes", "sunglasses"]`
- `--products-per-category`: 每个品类随机选取的商品数，默认 5
- `--max-reviews`: 每个商品处理的评论数，默认 25
- `--output-dir`: 实验结果输出目录
- `--seed`: 随机种子，默认 42

### 输出文件

- `experiment_log.jsonl` — 每条运行记录的 JSONL
- `experiment_summary.json` — 汇总统计
- 每个商品的完整 analysis artifact（JSON）保存在 `02_outputs/4_reports/{category}/` 下

## 运行 LLM-as-Judge 评估

实验完成后，运行评估：

```bash
.venv/bin/python -m decathlon_voc_analyzer.workflows.llm_judge_evaluation \
    --experiment-dir ./experiment_results \
    --output-dir ./evaluation_results
```

### 评估指标

**检索质量（Retrieval）**
- `retrieval_relevance_mean`: LLM 评判的证据相关性均值（1-5）
- `retrieval_coverage`: 至少有一条相关证据的问题比例

**报告质量（Report）**
- `report_coverage`: 报告对 aspect 的覆盖度（1-5）
- `report_factual_consistency`: 报告与证据的事实一致性（1-5）

**归因质量（Attribution）**
- `claim_support_rate`: supported claim / 总 claim
- `claim_grounded_rate`: (supported + partial) / 总 claim
- `citation_precision`: 有 product evidence 支撑的 claim / 有 citation 的 claim

### 输出文件

- `evaluation_log.jsonl` — 每条评估记录的 JSONL
- `evaluation_summary.json` — 按条件聚合的指标均值

## 快速测试（仅跑 1 个商品）

如需快速验证流程，可修改 `experiment_runner.py` 中的 `EXPERIMENT_CONDITIONS` 或限制运行数量：

```bash
# 仅跑 backpack 品类的前 1 个商品
.venv/bin/python -m decathlon_voc_analyzer.workflows.experiment_runner \
    --categories backpack \
    --products-per-category 1 \
    --max-reviews 10 \
    --output-dir ./test_run
```

## 注意事项

1. **API 费用**：8 个条件 × 15 个商品 = 120 次完整分析，每次分析包含多次 LLM 调用，请确认 API 预算充足。
2. **运行时间**：单个商品完整分析约需 2-5 分钟（取决于网络延迟），全部 120 次运行预计需要 4-10 小时。
3. **缓存**：实验运行器会复用已存在的 extraction artifact，但不会复用 analysis checkpoint（每次条件不同）。
4. **对照实验**：Lewis2020、JARVIS、VeriCite 的基线实现使用与主系统相同的 embedding 和 LLM gateway，仅改变检索策略和报告生成逻辑，确保公平比较。
