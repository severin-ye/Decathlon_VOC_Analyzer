# 数据集与商品证据标准化模块

## 模块定位

本模块对应 `stage1_dataset`，核心实现位于 `dataset_service.py` 和 `chinese_dataset_service.py`。它负责扫描 `01_data/01_raw_products/products/` 下的类目与商品目录，读取每个商品的 `product.json`、`reviews.json` 和 `images/`，并将原始电商数据统一转化为 `ProductEvidencePackage`。该证据包是后续评论建模、索引构建、检索、报告生成和评估的共同输入。

从论文角度看，该模块不是普通的数据读取层，而是整个系统的“证据建模层”。它把商品页面中的标题、描述、类目、评论和图片都转为带有稳定 ID、来源字段和语言标记的对象，使生成模型后续不能只基于自由文本推断，而必须围绕可追踪证据工作。

## 主要创新点

1. 面向 VOC 分析的商品证据包建模：系统没有把商品数据简单拼接成一段 prompt，而是显式区分 `TextEvidence`、`ImageEvidence`、`ImageRegion` 和 `ReviewRecord`，并将它们聚合为 `ProductEvidencePackage`。这种对象化建模让商品文案、商品图片和用户评论在同一证据空间中被统一调度。
2. 原始图像与局部区域的预留式建模：当前实现不仅记录整图，还为每张图生成 `center_focus`、`upper_focus`、`lower_focus`、`left_focus`、`right_focus` 五类默认区域。这使系统即使在没有人工标注框或检测模型的情况下，也能初步支持区域级视觉证据召回。
3. 可复现的证据 ID 体系：文本块、图片和区域都使用 SHA1 摘要生成稳定标识，避免在多次运行、缓存、评估和报告归因中出现对象漂移。
4. 多语言轻量识别：系统通过字符范围识别中文、韩文、希腊文、俄文和拉丁文字，为后续多语言召回和语言均衡候选池提供基础字段。
5. 数据质量显式输出：`build_overview()` 和 `normalize_dataset()` 会统计缺失文件、评论数量、图片数量、空评论数量和标准化产物路径，使论文实验可以说明数据完整性控制过程。

## 具体实现

`DatasetService.build_overview()` 首先扫描类目目录，统计每个类目下商品数、存在 `product.json` 的商品数、存在 `reviews.json` 的商品数、评论总数和图片总数。如果发现商品缺少产品文件或评论文件，系统会生成 warning，而不是静默跳过。

`DatasetService.normalize_dataset()` 根据请求中的 `categories`、`product_ids` 和 `max_products` 选择商品目录。对每个商品，系统调用 `_normalize_product()` 读取原始 JSON，清洗文本，构建文本块、图片证据和评论记录。如果请求开启 `persist_artifacts`，标准化后的商品证据包会写入 `02_outputs/1_normalized/<category>/<product_id>.json`，并额外生成数据标准化报告。

文本证据由 `_build_text_blocks()` 生成，当前包括三个来源：商品名称、商品描述和类目文本。每个文本块保存 `text_type`、`source_section`、`content_original`、`content_normalized` 和 `language`。这使后续检索不只知道命中了什么文本，也知道命中的文本来自标题、描述还是类目字段。

图像证据由 `_build_images()` 和 `_inspect_image_regions()` 生成。系统从 `product.json` 的 variants 中读取颜色变体和图片路径，记录图片宽高，并为有效图片创建默认区域。区域框由相对比例映射为真实像素坐标，例如中心区域使用 `(0.2, 0.2, 0.8, 0.8)`。后续 CLIP 图像 embedding 和多模态 reranker 可以对整图或区域裁剪执行编码和评分。

评论证据由 `_build_reviews()` 生成。系统为每条评论创建形如 `<product_id>_review_0001` 的稳定 ID，保留评分、正文、语言提示和空评论标记。该记录随后进入评论抽样和方面抽取模块。

## 与文献思路的关系

多模态 RAG 研究普遍强调，系统效果受限于前置解析与证据组织质量，而不仅取决于最终生成模型能力 [1][2]。本模块吸收这一思想，将商品页面拆解为结构化证据包，而不是直接让大模型从原始 JSON 中自由归纳。

原始模态检索与视觉文档检索研究强调保留图像原始形态，并逐步从页面级、对象级走向区域级证据 [3][4]。本项目没有直接采用 ColPali 式多向量 late interaction，但在商品图像层面保留整图和区域框，为后续区域级召回留出接口。

## 可写入论文的表述

本文首先构建面向电商 VOC 的商品证据包。不同于将商品信息直接拼接到提示词中的方法，本文将商品标题、商品描述、类目、评论、图片及图片局部区域统一建模为带有稳定标识的证据对象。该设计使后续评论理解、图文召回、证据归因和实验评估都可以基于同一对象体系完成，从而提高系统的可解释性和可复现性。
