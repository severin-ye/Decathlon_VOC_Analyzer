# 重排缓存与本地推理模块

## 模块定位

本模块对应 `stage3_retrieval/reranker_service.py`、`retrieval_cache_service.py`、`local_model_utils.py`、`openvino_models.py` 和 `openvino_integrated.py`。它位于粗召回之后，负责将候选商品证据重新排序，并通过缓存和本地推理能力降低重复运行成本。

论文中可将该模块定义为“高置信证据选择层”。embedding 粗召回负责扩大候选范围，reranker 负责判断候选是否真正能回答当前评论问题。

## 主要创新点

1. 文本与图像候选分离重排：系统先把候选按 `route` 分成文本和图像，再分别调用文本 reranker 或多模态 reranker。这样可以避免把视觉证据强行转成文本后丢失图像信息。
2. 多级重排后端：文本候选支持本地 Qwen3 reranker、DashScope `gte-rerank-v2` API 和启发式排序；图像候选支持本地 Qwen3-VL、多模态 Qwen-VL API、文本 reranker 退路和启发式排序。
3. 图像候选原图输入：多模态重排时，系统会读取商品图片，必要时按 `region_box` 裁剪，再压缩为 JPEG data URL 输入 Qwen-VL。这比只使用图片路径或代理文本更接近真实视觉判断。
4. 签名绑定缓存：query embedding 和 rerank 结果均写入磁盘缓存，缓存签名绑定 route、query、候选 digest、backend、model 和 base_url，避免不同配置下缓存互相污染。
5. 严格运行策略：当运行策略禁止降级时，embedding 或 reranker 失败会抛出 `RuntimePolicyError`，而不是静默退回 hash 或 heuristic。这使论文实验可以区分“真实模型结果”和“降级验证结果”。
6. 本地模型与 OpenVINO 预留：项目依赖和源码中已包含本地 Qwen3 embedding、reranker、Qwen3-VL 和 OpenVINO 相关入口，为后续低成本部署和边缘推理提供基础。

## 具体实现

`RerankerService.rerank()` 接收 query 和候选证据列表，先拆分为 `text_candidates` 和 `image_candidates`。文本候选进入 `_rerank_text_candidates()`，图像候选进入 `_rerank_image_candidates()`。最后系统把两路结果合并，并按 score 从高到低排序。

文本 API 重排通过 DashScope rerank endpoint 完成。系统发送 query 和候选文本，读取返回的 `relevance_score`，再将分数写回 `IndexedEvidence.score`。本地文本重排则通过 `get_reranker_model()` 加载 Qwen3-Reranker，并调用本地模型的 `rerank()`。

图像 API 重排通过 Qwen-VL chat completion 完成。系统为每个候选构造文本提示，并把对应图片或裁剪区域作为 image_url 内容块输入模型，要求模型返回严格 JSON：`{"results":[{"index":0,"relevance_score":0.0}]}`。该过程让视觉模型直接观察商品图，而不是只依赖图像文件名或商品文案。

缓存由 `_rerank_with_cache()` 封装。缓存签名通过 `candidate_digest` 绑定候选集合，因此即使 query 相同，只要候选集发生变化，系统也会重新计算重排结果。query embedding 缓存同样由 `EmbeddingService` 和 `RetrievalCacheService` 管理，内存中保留 LRU，磁盘上保留可复用文件。

## 与文献思路的关系

两阶段检索系统通常将召回与精排分离，以在成本和质量之间取得平衡 [1][2]。本项目延续该思想，先使用 embedding 找到较宽候选，再用 reranker 提高证据排序质量。

多模态 RAG 研究强调，图像证据不应在过早阶段被完全转写为文本，否则会损失视觉结构信息 [3]。本模块在图像重排阶段直接输入原图或区域裁剪，使视觉证据能够参与最终证据排序。

结构化 JSON 输出约束能够提高 LLM 作为系统组件时的可控性 [4]。本模块要求 Qwen-VL reranker 返回固定 JSON 结构，并在解析失败时显式报错或降级。

## 可写入论文的表述

本文在粗召回之后引入模态感知重排机制。文本候选和图像候选分别由适配其模态的 reranker 处理，其中图像候选可直接输入原始商品图或局部裁剪区域。为保证实验可复现，系统将 query embedding 与 rerank 结果按照模型、后端和候选集合签名进行缓存，并通过运行策略区分严格模式与降级模式。
