# 评估模块与实验指标

## 模块定位

本模块对应 `evaluation/manifest_evaluation_service.py`。它读取批处理运行产生的 manifest、analysis artifact、feedback sidecar 和 replay sidecar，计算检索质量、评论建模质量、报告证据归因和回放状态等指标。

论文中可将该模块定义为“实验评估层”。它使系统不只输出案例报告，还能对检索和证据支撑进行量化分析。

## 主要创新点

1. manifest 驱动评估：评估入口不是临时读取某个 JSON，而是从 run manifest 中发现 analysis、feedback 和 replay 产物，保证评估与实际运行绑定。
2. 支持显式 gold labels：如果 manifest 或 analysis 中包含 `retrieval_relevance` 标签，系统可计算 Recall@1、Recall@3、Recall@5、MRR、NDCG@3 和 NDCG@5。
3. 无标签质量统计：即使没有人工 gold labels，系统仍统计 informative review 数、aspect 数、question 数、retrieval 数、平均证据数量、证据覆盖率、分数漂移和冲突风险。
4. Claim attribution 指标：系统统计 supported、partial、unsupported、contradicted claim 的数量与比例，计算 claim support rate、claim grounded rate、citation precision、citation contradiction rate 和 route contribution。
5. 反馈与回放评估：系统读取 feedback 和 replay sidecar，统计 persistent issues、applied corrections 等信息，支持论文讨论人机闭环效果。

## 具体实现

`discover_manifest_paths()` 可接收单个 manifest 文件或目录。如果是目录，则递归查找 `*_run_manifest.json`。`load_bundle()` 读取 manifest，并根据其中的 `artifact_bundle` 字段加载 analysis、feedback 和 replay JSON。

`_extract_retrieval_labels()` 从 `manifest.evaluation_labels.retrieval_relevance` 或 `analysis.retrieval_labels` 中读取人工标注。每条标注可以提供 `relevant_evidence_ids`，也可以提供 `graded_relevance`。如果只提供相关证据 ID，系统默认相关度为 1.0。

`_build_formal_retrieval_metrics()` 遍历每条 retrieval，按 `retrieval_id` 或 `source_question_id` 对齐标签，然后计算 Recall@K、MRR 和 NDCG。证据 ID 优先使用 region_id、text_block_id、image_id 或 evidence_id。

`_build_claim_attribution_metrics()` 从报告的 `claim_attributions` 中统计证据支撑状态。它排除 evidence_gap 来源的 claim，计算 supported、partial、unsupported、contradicted 的分布，并进一步统计 route_sources 中 text、image 和 mixed 的贡献比例。

`evaluate_bundle()` 汇总所有指标，包括评论、方面、问题、检索、报告和回放。该结果可作为论文实验表格的基础数据。

## 与文献思路的关系

信息检索领域常用 Recall@K、MRR 和 NDCG 衡量召回和排序质量 [1]。本项目将这些指标用于多模态商品证据检索，评估问题级召回是否命中了人工标注证据。

RAG 评估除了答案质量，也需要关注引用证据是否真正支持生成主张 [2]。本模块通过 claim attribution 指标衡量报告主张的证据支撑情况。

RARR 等研究强调对模型输出进行后验校正与验证 [3]。本项目通过 feedback/replay 指标记录问题是否持续存在以及修正是否被应用，为迭代式评估提供数据基础。

## 可写入论文的表述

本文从检索质量和生成证据支撑两个层面评估系统。对于带人工标签的检索问题，系统计算 Recall@K、MRR 和 NDCG；对于生成报告，系统统计 claim 级支撑状态、引用精度和模态贡献比例。该评估设计能够同时回答“系统是否找到了正确证据”和“系统生成的结论是否被证据支持”两个问题。
