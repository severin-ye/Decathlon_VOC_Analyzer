# Aspect级评论建模

## 本层目标

把评论从“整段自然语言”转成“可被统计、召回、聚合、引用”的结构化对象。

## 抽取对象

每条评论至少拆成：

- aspect
- sentiment
- opinion
- evidence_span
- usage_scene
- confidence

## 为什么 aspect 级建模是核心

对话中反复强调，项目不是做简单正负分类，而是做商品印象与客户印象建模。只有先得到方面级结构，后续才能进行：

- 商品优缺点统计
- 使用场景聚合
- 客户需求聚合
- 改进建议约束生成

## 模型职责

ChatQwen 在这一层的角色不是生成结论，而是稳定输出 structured output。

## 本层输出用途

- 驱动“待澄清问题生成”层
- 间接驱动商品多模态召回
- 驱动商品印象与客户印象聚合
- 为最终生成提供可引用原句

## 与召回层的边界

本层产物不应被直接当成最终召回 query。

更合理的边界是：

- 评论建模层负责给出 aspect、opinion、evidence_span、usage_scene
- 问题生成层负责把这些结构化对象改写为若干待验证问题
- 召回层只对问题做检索，不直接对评论原文检索

这样能让评论建模层保持“抽取”职责，召回层保持“找证据”职责，中间的“问题化”成为专门一层。
