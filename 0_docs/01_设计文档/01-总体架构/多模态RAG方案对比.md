# 多模态RAG方案对比

## 目的

这篇文档专门整理原始对话中对多模态 RAG 三种主流落地方案的比较，避免这些背景判断在项目文档中被压缩丢失。

## 方案 A：Text-first

### 核心思路

先把图片、表格、图表、视频等内容尽量转成文本，再走传统文本 RAG。

### 典型做法

- 图片转 OCR + caption
- 表格转 Markdown 或 HTML
- 图表转描述文本
- 视频转 ASR 和关键帧说明

### 适合场景

- 快速原型
- 成本有限
- 视觉细节要求不极端

### 局限

- 视觉细节损失明显
- 图像和图表语义高度依赖转写质量

## 方案 B：Dual-store

### 核心思路

同时保留原始模态与文本化摘要，在检索和生成阶段按需组合使用。

### 典型做法

- 图像保留原图，同时保存 caption 或 OCR 摘要
- 表格保留结构化表示，同时保存文本转写
- 检索阶段走双路召回
- 生成阶段只把关键原始证据送入模型

### 适合场景

- 企业级文档问答
- 商品图文分析
- 医疗影像与报告联合分析

### 局限

- 设计复杂度高于 Text-first
- 需要处理模态间对齐和合并

## 方案 C：Native Multimodal

### 核心思路

- 检索时就使用多模态 embedding
- 生成时直接使用多模态大模型
- 全流程支持 text、image、video、audio 等多模态输入

### 适合场景

- 高预算项目
- 研究型项目
- 对图表、视觉证据要求高的任务

### 局限

- 工程复杂
- 成本高
- 调优难

## 对当前项目的结论

对话中的最终判断不是“全盘采用纯粹的 Native Multimodal”，而是：

- 在商品证据召回上采用 Native Multimodal 倾向
- 在评论理解上采用结构化文本建模
- 在最终生成上采用证据约束的多模态总结

因此，当前项目更准确地说是“以 Native Multimodal 为召回核心的混合多模态系统”。
