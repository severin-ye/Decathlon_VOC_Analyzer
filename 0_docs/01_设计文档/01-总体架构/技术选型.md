# 技术选型

## 目标选型

对话中最终收束出的推荐组合如下：

- 编排层：LangChain
- 生成层：ChatQwen + Qwen VL 系列
- 召回层：Qwen3-VL-Embedding
- 精排层：Qwen3-VL-Reranker
- 向量库：Qdrant
- 文档解析扩展：Docling

## 角色分工

### LangChain

- 编排整体链路
- 组织多模态消息
- 处理 structured output
- 承接工具调用

### Qwen VL 生成模型

- 阅读最终证据包
- 输出综合意见
- 输出改进建议
- 输出结构化结果

### Qwen3-VL-Embedding

- 统一编码商品图像与商品文本
- 为 query 与商品证据建立共享语义空间
- 支持初始召回

### Qwen3-VL-Reranker

- 对商品图与商品文本的候选证据做精排
- 降低 embedding 粗召回带来的噪声

### Qdrant

- 保存图像与文本索引
- 支持后续向多向量或更复杂 schema 扩展
- 适合做生产级召回与评估

## 为什么不选择纯 OpenClaw 路线

对话中的判断是：

- OpenClaw 本体不是多模态 RAG 框架核心。
- ClawRAG 当前更偏向 Docling + 文本或混合检索。
- 它不是当前项目所需的原始模态召回主线。

## 模型使用建议

- 原型优先：较轻的 embedding + API 生成模型
- 研究导向：本地 embedding 与 reranker，生成模型可 API
- 高质量阶段：引入 region-level 证据和更强 reranker
