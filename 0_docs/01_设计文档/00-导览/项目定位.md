# 项目定位

## 项目目标

本项目不是普通的问答式 RAG，也不是单纯的情感分析，而是一个面向迪卡侬商品场景的证据驱动多模态商品洞察系统。

系统目标包括四层：

- 对商品图片与商品文字介绍进行统一召回。
- 依据评论内容构建商品印象与客户印象。
- 基于证据生成客户对商品的综合意见。
- 提出可以追溯到证据来源的商品改进建议。

## 核心判断

对话中已经明确，本项目最适合的路线不是“纯原始模态端到端黑箱生成”，而是“分阶段保真”的系统设计：

- 商品侧尽量保留原始模态。
- 评论侧尽快转成结构化证据。
- 聚合与生成阶段严格受证据约束。
- 每个阶段都保留可解释性。

## 系统本质

项目可以概括为一句话：

一个面向电商商品分析的证据驱动多模态洞察系统，将商品图片与商品文案进行统一召回，以评论中的方面级证据为桥梁，构建商品印象与客户需求印象，并在保留全链路可解释性的前提下生成综合意见与改进建议。

## 为什么不能只做纯文本方案

- 商品图中的结构、材质、场景、细节会影响用户评价。
- 仅靠评论文本，无法解释某些观点与商品视觉证据的对应关系。
- 仅靠商品描述文本，难以承接“缓震、透气、外观、结构”这类视觉相关判断。

## 为什么也不能完全依赖纯视觉方案

- 评论证据主体仍然是文本。
- 商品改进建议需要综合评论句子、商品文案、商品图像三类证据。
- 全流程都交给视觉大模型会降低可解释性，并提高成本与调试难度。
