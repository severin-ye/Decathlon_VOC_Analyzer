# 综合意见生成

## 输入约束

综合意见不能直接基于原始全量数据生成，而应基于已经组织好的证据包：

- 商品图像证据
- 商品文案证据
- 评论方面级证据
- 聚合统计结果
- 代表性评论

## 输出目标

综合意见应覆盖以下内容：

- 商品总体评价
- 主要优点
- 主要缺点
- 争议点
- 适用场景或适用人群

## 生成原则

- 先统计，再总结
- 先筛证据，再生成
- 结论必须对应至少一类可追溯证据

## 为什么不能直接自由生成

如果没有证据约束，系统会失去两类能力：

- 结果复盘能力
- 误差定位能力

## 推荐输出格式

建议结构化输出：

- answer
- supporting_aspects
- supporting_reviews
- supporting_product_evidence
- confidence
