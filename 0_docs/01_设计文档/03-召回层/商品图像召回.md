# 商品图像召回

## 召回对象

图像召回覆盖的对象包括：

- 主图
- 细节图
- 场景图
- 结构图
- 后续可扩展的局部裁剪图

## 建议模型

使用 Qwen3-VL-Embedding 对图像建立向量表示。

## 在线查询来源

查询通常来自评论方面对象，例如：

- 缓震好
- 夏季闷
- 外观好看
- 收纳分区少

## 在线查询逻辑

1. 从评论方面对象生成简洁 query。
2. 将 query 编码为向量。
3. 在商品图像索引中做相似度搜索。
4. 返回候选 image_id 及其得分。

## 这一层解决的问题

- 让评论中的观点能对应到商品视觉证据。
- 为后续“结构改进建议”提供图像支撑。

## 后续扩展

对话中提出了更强的方向：

- page-level 到 region-level 的演进
- 局部裁剪图的引入
- 更细粒度的视觉证据定位
